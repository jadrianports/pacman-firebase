# Phase 7 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Scope note:** A public, mobile-first, read-only Firebase Hosting page (WEB-01/02/03) that is a pure
> consumer of the live `get_leaderboard` API and mirrors the in-game This Week / All Time boards in full
> retro arcade style. Plain static HTML/CSS/JS — no framework, no build step (D-01). The shadcn gate does
> **not** apply (not a React/Next/Vite project), so `Tool: none` and registry safety is not applicable.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (plain static HTML/CSS/JS, no build step — D-01) |
| Preset | not applicable |
| Component library | none |
| Icon library | none — inline SVG only (refresh glyph + Pac-Man favicon); honors no-new-deps ethos |
| Font | **Press Start 2P** (pixel webfont, headers/title only — D-06) + system **monospace** fallback for body/score/rank rows |

**Visual direction (locked, D-04/D-05):** Full retro arcade — black background, Pac-Man-yellow accents,
chunky rank list, arcade chrome. The concrete visual build is a **hard directive to route through the
`frontend-design` skill** (D-05) — distinctive and intentional, not a generic dark-theme template. The
tokens below are the contract `frontend-design` must honor; it may add arcade chrome/detail within them.

**Font rationale:** Press Start 2P is single-weight (400) and unreadable at small sizes, so it is reserved
for the `PAC-MAN` wordmark and the tab labels only. Rank rows (initials, scores) use a **monospace** stack
so the dot-leader layout (`1. BOB ...... 12345`) aligns column-perfectly and small numbers stay legible on
a phone. Monospace stack: `ui-monospace, "SF Mono", "Cascadia Mono", Menlo, Consolas, monospace`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Dot-leader / inline gaps inside a rank row |
| sm | 8px | Gap between rank rows, tab-label inner padding |
| md | 16px | Default element spacing, board container side padding (mobile gutter) |
| lg | 24px | Gap between header block, tab bar, and the board |
| xl | 32px | Space above/below the PAC-MAN title |
| 2xl | 48px | Top-of-page header block, major section break |
| 3xl | 64px | Reserved — desktop max-width centering / page-level breathing room |

Exceptions: interactive controls (the `This Week | All Time` tabs and the `Refresh` control) must have a
**minimum 44px touch-target height** for mobile (WCAG 2.5.5 / Apple HIG), even when their text box is
smaller. This is the only allowed deviation from the 8-point scale.

---

## Typography

Mobile-first. Exactly two weights in the scalable (monospace) family; Press Start 2P is inherently 400-only.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display (`PAC-MAN` wordmark) | 28px | 400 (Press Start 2P) | 1.2 |
| Heading (tab labels: `This Week` / `All Time`) | 16px | 400 (Press Start 2P) | 1.2 |
| Body (rank rows: rank + initials + score) | 16px | 400 / **700** for rank-1 row (monospace) | 1.5 |
| Label (last-week subtitle, hint, empty/offline lines) | 12px | 400 (monospace) | 1.5 |

## Color

60/30/10 split. Dominant black + Pac-Man yellow accent is the locked direction (D-04); echoes
`settings.py` (`COLOR_YELLOW=#FFFF00`, `COLOR_WHITE=#FFFFFF`, `COLOR_GRAY=#808080`).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #000000 | Page background (full retro arcade) |
| Secondary (30%) | #10102E | Board container panel, tab-bar surface — deep arcade navy giving "cabinet screen" depth without competing with yellow |
| Accent (10%) | #FFFF00 | See reserved-for list below |
| Destructive | not applicable | Read-only page — no destructive actions exist |

**Accent (#FFFF00) reserved for:** (1) the `PAC-MAN` wordmark title, (2) the **active** tab label/indicator,
(3) the **rank-1** leaderboard row, (4) the `Refresh` control's active/focus highlight. Nothing else.

**Supporting (non-accent) text colors** — mirror the in-game board (`menu.py:run_leaderboard`):

| Color | Value | Usage |
|-------|-------|-------|
| White | #FFFFFF | Rank rows 2–10 (initials + score) |
| Gray | #808080 | Hints, last-week subtitle, **inactive** tab label, empty/offline lines, dot-leader characters |

## Copywriting Contract

All strings mirror the in-game board verbatim for parity (D-12, `menu.py:run_leaderboard`).

| Element | Copy |
|---------|------|
| Primary CTA | **Refresh** — re-pulls the currently active view (D-10); the only interactive action besides the tab toggle |
| Empty state — This Week | `No scores yet this week. Be the first!` |
| Empty state — All Time | `No scores yet. Be the first!` |
| Error state | `Could not connect to leaderboard.` — solution path: the visible **Refresh** control re-attempts the active view (each view degrades independently; the other cached view still renders) |
| Destructive confirmation | not applicable — no destructive actions |

**Additional locked strings:**

| Element | Copy |
|---------|------|
| Loading | `Loading...` (mirror in-game) |
| Tab labels | `This Week` \| `All Time` (default view on load = **All Time**, deliberate web divergence — D-08) |
| Last-week champion subtitle | `Last week: {INITIALS}` — This Week view only; **hidden when last week has no champion** (D-09) |
| Rank row format | `{rank}. {INITIALS}  {dot-leaders}  {score}` — rank-1 in yellow, 2–10 in white; consider thousands separators on score (Claude's discretion, D) |
| Browser tab title | `PAC-MAN — Leaderboard` (D-14) |
| OG / Twitter Card | title `PAC-MAN — Leaderboard`, arcade description, static ~1200×630 preview image (D-13) |
