<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → get_leaderboard (HTTP) | Untrusted query string (`?scope=`) crosses here; the function is a public read endpoint. |
| get_leaderboard → Firestore | Server-controlled query; week_id is computed server-side, never taken from the client. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | `?scope=` query param | mitigate | Tolerant allow-list parse: only `week`/`all`/`last_week` are honored; anything else silently falls back to `week`, never a 400 (D-01). No client value reaches the Firestore query except via the closed allow-list. |
| T-06-02 | Information disclosure | last_week response projection | mitigate | Reuse the existing projection loop (lines 62-68) — only `{initials,score}` ship; machine_id/week_id/updated_at never leave the server (existing control from Phase 4 D-10, relied on unchanged). Verified by `test_scope_last_week_projects_only_initials_and_score`. |
| T-06-03 | Spoofing | forged week selection | mitigate | The queried week is `previous_week_id(current_week_id())` computed from SERVER time only — the client cannot request an arbitrary week_id (inherits Phase 4 D-06 server-time-only control). |
| T-06-04 | Denial of service | request flooding on the read endpoint | accept | Public read of `{initials,score}` top-10 only; `--max-instances=5` cap already set on the service (Phase 4 D-11). No new attack surface vs. the existing `week`/`all` scopes. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (RESEARCH "Package Legitimacy Audit": N/A — stdlib + already-vendored firebase_admin/functions_framework only). |

Existing controls relied upon (do NOT re-litigate): HMAC signing + server verification on submit (Phase 4 COMP-01), score sanity ceiling (COMP-02), permanent server-side initials (COMP-03), {initials,score}-only projection (D-10). This read-only branch adds no write surface.
</threat_model>
