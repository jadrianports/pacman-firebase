# Phase 6 — UI Design Contract

> Visual and interaction contract for the in-game weekly boards (This Week / All Time toggle, last-week champion subtitle) and the launch got-passed banner. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Medium note:** This is a PyGame desktop arcade game rendered to a 900x950 surface — NOT a web/React UI. There is no DOM, no CSS, no component library, no shadcn. "Spacing" = pixel y-offsets between centered blits; "Typography" = `pygame.font.Font` sizes; "Color" = RGB tuples in `settings.py`. The contract below is expressed in the game's existing PyGame idiom so the executor can implement it directly against `menu.py` / `main.py` / `settings.py`. Every value is pinned to the established visual language already shipped in `run_main_menu` / `run_leaderboard` / `run_game_over_screen`.

## Spacing Scale

The game uses absolute centered y-coordinates, not a margin/padding token system. To satisfy the "multiples of 4" design-quality dimension while staying faithful to the existing arcade layout, vertical rhythm is declared on a **4px grid**. Existing screens already align to this grid (title y=150, options y=350 step 70, header y=80, entries y=180 step 50, hints y=HEIGHT-80 → 870 — all multiples of 4 except where noted).

| Token | Value | Usage (this phase) |
|-------|-------|--------------------|
| step-xs | 8px | Gap between tab-indicator glyphs and labels (inline kerning slack) |
| step-sm | 16px | Subtitle baseline offset tuning |
| row | 48px | Leaderboard entry row pitch (existing screen uses 50px; this phase keeps the existing 50px row pitch for byte-faithful continuity — see Exceptions) |
| block | 24px | Separation between header and the last-week subtitle |
| section | 64px | Major vertical breaks (header band → board band) |

**Declared y-anchors for new/changed elements (900x950 canvas, all centered on x=WIDTH//2=450):**

| Element | y | Notes |
|---------|---|-------|
| LEADERBOARD header (unchanged) | 80 | existing |
| Tab indicator `< This Week \| All Time >` (NEW) | 128 | sits between header (80) and subtitle/first entry; 48px below header |
| Last-week champion subtitle (NEW, This Week only) | 152 | 24px below tab indicator, 28px above first entry; hidden on All Time and when absent (D-16) |
| First board entry (unchanged) | 180 | existing; rows step +50 (i\*50) |
| Offline / empty-state line (unchanged) | HEIGHT//2 = 475 | existing center; 475 is on the 4px-off; kept as-is for continuity (Exception) |
| Back hint (unchanged) | HEIGHT-80 = 870 | existing |
| Main-menu PAC-MAN title (unchanged) | 150 | existing |
| Got-passed banner line (NEW, main menu) | 230 | between title (150) and first option (350); 80px below title, 120px above options — clear of both |
| Main-menu options (unchanged) | 350 + i\*70 | existing |

Exceptions:
- **Entry row pitch stays 50px** (not 48) — preserves the existing `run_leaderboard` rendering byte-for-byte so the All Time view looks identical to today's shipped board (D-19 "keep current styling"). Changing it would be a gratuitous regression of a working screen.
- **Center-anchored degrade lines stay at HEIGHT//2 (475)** — existing offline/empty pattern (`menu.py:141,145`); kept for continuity with `run_game_over_screen`.
- These two exceptions are deliberate continuity-with-shipped-UI choices, not new arbitrary values.

## Typography

Three font sizes only — the game's existing scale. No new sizes introduced (satisfies "3-4 sizes" and avoids font-asset bloat per RESEARCH A4). Weight is intrinsic to `freesansbold.ttf` (a single bold face) — there is no variable-weight axis in PyGame bitmap font rendering, so "weight" is fixed bold for all roles; line-height is not applicable (single-line centered blits, no text wrapping).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display (PAC-MAN title, GAME OVER, big letters) | 64px (`FONT_TITLE`) | bold (freesansbold, fixed) | n/a (single line) |
| Heading (LEADERBOARD, menu options, score) | 36px (`FONT_MENU`) | bold (fixed) | n/a |
| Body/Label (board entries, tab indicator, subtitle, banner, hints) | 24px (`FONT_SMALL`) | bold (fixed) | n/a |

**Role assignments for new elements:**
- **Tab indicator** `< This Week | All Time >` → 24px (`FONT_SMALL`). Rationale: it is a secondary control sitting under the 36px header; 36px would compete with the header for emphasis.
- **Last-week champion subtitle** `Last week: BOB` → 24px (`FONT_SMALL`). It is a subtitle, intentionally subordinate to the LEADERBOARD header (D-04 "small subtitle").
- **Got-passed banner** → 24px (`FONT_SMALL`). Prominent by color and position, not by size; must fit one line at WIDTH=900 (drives the cap, see Copywriting).

Line-fit budget: at 24px `freesansbold`, the banner and tab indicator must each fit within ~860px (900 minus ~20px side breathing room). This budget drives the banner name cap below.

## Color

The game's palette is five fixed RGB tuples in `settings.py`. The 60/30/10 split is mapped to the arcade medium: black is the dominant surface, white is the secondary/default text, yellow is the reserved accent. No new colors are added — reusing the shipped palette keeps the new UI visually identical in character to existing screens (a design-consistency requirement, not just convenience).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `(0,0,0)` black (`screen.fill("black")`) | Full-screen background on every menu/board frame |
| Secondary (30%) | `(255,255,255)` `COLOR_WHITE` | Default text: board entries ranks 2-10, inactive tab label, score, menu options (unselected) |
| Accent (10%) | `(255,255,0)` `COLOR_YELLOW` | Reserved — see explicit list below |
| Muted (supporting) | `(128,128,128)` `COLOR_GRAY` | Subordinate/degrade text: hints, "Could not connect", empty-state lines, last-week subtitle |
| Semantic (status) | `(255,0,0)` `COLOR_RED` / `(0,255,0)` `COLOR_GREEN` | Existing game-over/victory only — NOT used by any Phase 6 element |

**Accent (yellow) reserved for — explicit, closed list:**
1. Rank-1 board entry (existing, unchanged — D-19).
2. Section headers: LEADERBOARD header, PAC-MAN title (existing, unchanged).
3. The **active** tab label in `< This Week | All Time >` (NEW) — the active side is yellow, the inactive side is gray; the `<`, `|`, `>` separators are gray.
4. The **got-passed banner** (NEW) — yellow, to read as an urgent "go look" prompt against the black menu, consistent with yellow = "this is the important thing" already established by the title and rank-1.

Yellow is NOT used for: every board row (only rank 1), inactive tab, banner separators, hints, subtitles, or any "all interactive elements" blanket. This keeps yellow at roughly 10% of on-screen ink.

**Last-week champion subtitle color:** GRAY (`COLOR_GRAY`), not yellow. Rationale: it is a subtitle subordinate to the header (D-04 "small subtitle"), and reserving yellow for the rank-1 row + banner preserves the 10% accent discipline. Gray also matches its sometimes-absent, low-stakes nature (D-16).

**Destructive:** none. This phase has **no destructive actions** (read-only board viewing + a banner). The marker file rewrite on board-open (D-10) is silent local state, not a user-facing destructive action requiring confirmation.

## Copywriting Contract

All copy is terminal/arcade register (terse, ALL-CAPS for titles, sentence case for status lines) matching the shipped tone (`"No scores yet. Be the first!"`, `"Could not connect to leaderboard."`, `"Press ESC or ENTER to go back"`).

| Element | Copy |
|---------|------|
| Primary CTA | none — there is no submit/confirm button this phase. The closest "primary action" is the **toggle**, surfaced via the tab indicator + hint copy (below). The implicit call-to-action the banner creates is "open the Leaderboard," fulfilled by the existing main-menu "Leaderboard" option (no new label). |
| Tab indicator (active = This Week) | `< This Week \| All Time >` — active side rendered yellow, inactive gray. The `<` `>` are affordance hints that LEFT/RIGHT switch tabs. |
| Toggle hint (board screen) | `LEFT/RIGHT: switch board   ESC/ENTER: back` (replaces the current single `Press ESC or ENTER to go back` so the new toggle is discoverable; mirrors the `run_initials_entry` hint style on `menu.py:91`). Gray, 24px, at y=870. |
| Last-week champion subtitle | `Last week: {INITIALS}` (e.g. `Last week: BOB`). Shown only on This Week (D-04). Rendered from `entries[0]["initials"]` of the `scope=last_week` fetch. |
| Empty state — This Week | `No scores yet this week. Be the first!` (D-17) |
| Empty state — All Time | `No scores yet. Be the first!` (existing string, unchanged — D-17) |
| Last-week champion absent | render **nothing** — no placeholder, no dash, no "Last week: —" (D-16) |
| Error / offline state (per view) | `Could not connect to leaderboard.` (existing string, unchanged — D-15; applies to whichever view's fetch failed; the other cached view still renders) |
| Got-passed banner — single passer | `{INITIALS} passed you this week!` (e.g. `BOB passed you this week!`) |
| Got-passed banner — multiple, within cap | `{A}, {B}, {C} passed you this week!` (comma-separated initials) |
| Got-passed banner — over cap | `{A}, {B}, {C} +{K} more passed you this week!` (e.g. `JIM, BOB, ACE +2 more passed you this week!`) (D-06) |
| Got-passed banner — none / offline / cold-start | render **nothing** — no banner, no error (D-18, SC-4) |
| Destructive confirmation | n/a — no destructive actions this phase |

**Banner name cap (D-06, Claude's discretion → DECIDED): cap = 3 names.**
- At 24px `freesansbold` on a 900px canvas, `JIM, BOB, ACE +2 more passed you this week!` measures comfortably under the ~860px single-line budget; a 4th 3-char initial + ", " (`+~95px`) risks crowding the right edge on longer suffixes. Three names keeps every banner variant on one line with margin.
- Format rule for the executor: if `len(new_passers) <= 3`, list all joined by `, `. If `> 3`, list the first 3 joined by `, `, then ` +{len-3} more`. Always suffix ` passed you this week!`.
- Ordering of names is `sorted(new_passers)` (stable, deterministic — RESEARCH §6) so the banner is reproducible and testable.

**Loading state (unchanged):** `Loading...` (white, 36px, screen center) — existing, shown before the first board fetch resolves (`menu.py:121`).

## Interaction Contract (PyGame-specific — supplements the web-oriented template)

| Interaction | Binding | Behavior |
|-------------|---------|----------|
| Switch board view | `K_LEFT` / `K_RIGHT` | Flip `active` between `"week"` and `"all"`. First switch to All Time triggers a one-time lazy fetch showing `Loading...` then caching the result (D-14). This Week is the open-on view (D-02). |
| Exit board | `K_ESCAPE` / `K_RETURN` | Return from `run_leaderboard` (unchanged). Exiting also performs the marker baseline rewrite + banner clear (D-07/D-10) — a silent side effect, no visible confirmation. |
| Dismiss banner | (implicit) | Banner persists on the main menu across re-entries until the board is opened; opening the board clears it for the session and re-baselines for next launch (D-10). No explicit dismiss key. |
| Quit | `QUIT` event / existing menu nav | Unchanged. |

**State → render truth table for the board (drives the executor; resolves Pitfall 1):**

| View cache value | Renders |
|------------------|---------|
| `_UNFETCHED` sentinel | `Loading...` then fetch (never sits in this state across frames) |
| `None` | `Could not connect to leaderboard.` (gray, center) |
| `[]` (empty list) | empty-state string for the active view (This Week vs All Time wording) |
| `[...]` list | board rows; rank-1 yellow, rest white; last-week subtitle above (This Week only) |

**Graceful-degrade guarantees (SC-4, hard contract):**
- Offline at launch → no banner, no error, straight to menu (D-18).
- One view offline → that view shows the connect error; the other still renders if cached (D-15).
- Last-week subtitle fetch fails → subtitle simply absent, board otherwise normal (D-15/D-16).
- No spinner-hang: launch banner fetch uses `BANNER_FETCH_TIMEOUT_SECONDS` (~2s, Claude's discretion → **DECIDED: 2 seconds**), never the 10s default; a slow/dead network skips the banner silently (D-09).
