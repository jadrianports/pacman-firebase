## Cross-Phase Integration

Integration checker result: **0 blockers, 2 warnings** across the built phases (4-6). All five
critical seams are correctly wired:

1. ✅ Client→server signing↔verification loop (IDENT-03 ↔ COMP-01) — `canonical_message` byte-identical, HMAC-SHA256 + `compare_digest`, signature field flows submit→server gate.
2. ✅ Scoped API consumer (BOARD-03/04) — client sends `scope=week|all|last_week`; server has matching branches.
3. ✅ last_week week-id math — client `client_current_week_id` and server `current_week_id` are mirror Monday-UTC implementations; server owns `previous_week_id` (correct, server-time-only).
4. ✅ Marker re-baseline → banner (RIVAL-01) — submit path and board-open path both update the marker.
5. ⚠ Graceful degrade — works, but see CR-01 warning below.

**Warnings (tech debt, non-blocking):**
- **CR-01:** launch banner block (`main.py:101-109`) has no `try/except`; malformed `tracked_best`
  or a leaderboard entry missing `score` can crash startup. One-line fix.
- **RIVAL-01 browse-first edge:** a player who opens the board before ever playing never sets a
  baseline, so the banner never activates for them (no crash, no false positives).
