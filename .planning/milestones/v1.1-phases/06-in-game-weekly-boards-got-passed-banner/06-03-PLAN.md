<objective>
Create the unsigned last-viewed marker file IO (D-13) and the supporting settings constants. The marker is the deliberate INVERSE of the Phase-5 signed identity blob: plain JSON, no obfuscation, no HMAC, best-effort — a missing/corrupt/wrong-week marker is harmless and silently re-baselines. It holds the player's tracked this-week best and the set of initials above them at last board view, stamped with a client-computed Monday-UTC `week_id`.

Purpose: Plan 04 (launch banner + board-open baseline) consumes this marker to compute "who passed you since you last looked" (RIVAL-01, D-07/D-10/D-11/D-12). The client has no server `current_week_id` (confirmed: root `leaderboard_crypto.py` lacks it), so the week stamp is computed inline with stdlib `datetime`, mirroring the server's Monday-UTC math.
Output: new `marker.py` module, new `settings.py` constants, new `tests/test_marker.py`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local disk → game client | The marker file is local, user-writable, and UNSIGNED by design (D-13) — a player can edit it. |
| in-memory state → marker file | The marker is written best-effort; a write failure must not affect gameplay. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-08 | Tampering | the unsigned marker file | accept | Per D-13 the marker is intentionally unsigned/unobfuscated — re-signing on every view is pointless friction, and a wrong/edited marker is HARMLESS: the worst case is a wrong or suppressed got-passed banner (cosmetic). It controls no score, no submission, and no server state. Tamper-proofing it would be security theater (RESEARCH Anti-Patterns; Phase 5 D-03 "a wrong got-passed banner is harmless"). |
| T-06-09 | Denial of service | marker IO crashing startup | mitigate | All read/write wrapped in try/except → return silently (mirrors `_safe_remove`); a read-only %LOCALAPPDATA% or disk error can never raise into the startup path (Pitfall 5, SC-4). |
| T-06-10 | Tampering | malformed/corrupt marker on load | mitigate | `read_marker` returns None on FileNotFoundError/JSONDecodeError/any exception → caller treats as cold-start and re-baselines; corrupt data cannot crash or mislead beyond a one-time harmless re-baseline. |
| T-06-11 | Information disclosure | marker contents | accept | The marker stores only the player's own tracked best + initials already shown on the public board. No PII, no secret (the HMAC secret is NOT here — D-09), no other player's private data. Local-only. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs (RESEARCH "Package Legitimacy Audit": N/A — stdlib `json`/`os`/`datetime` only). |

Explicitly NOT applied (and why): obfuscation/HMAC on the marker — forbidden by D-13. Do not mirror the identity-blob signing path here.
</threat_model>
