## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Tampering | scope query-string assembly | mitigate | Build the query with `urllib.parse.urlencode` (escapes safely) rather than string concatenation, so a stray scope value cannot inject extra query params. The server independently allow-lists scope (Plan 01), so a forged scope only ever degrades to `week`. |
| T-06-06 | Tampering / Spoofing | forged or malformed leaderboard response (MITM / hostile endpoint) | mitigate | `get_leaderboard` wraps the whole fetch+parse in `try/except → None`; a malformed body raises and is swallowed to `None`, which the UI renders as the graceful "Could not connect" line (no crash, no leak). Display is read-only `{initials,score}`; a spoofed board cannot escalate (no client trust in board contents — your-best is tracked locally, Plan 04). |
| T-06-07 | Denial of service | a hung/slow endpoint blocking the client | mitigate | `urlopen` is always called with an explicit `timeout`; the launch banner (Plan 04) passes the short `BANNER_FETCH_TIMEOUT_SECONDS` so a dead network can never hang startup. Default 10s preserved for the interactive board screen. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs (RESEARCH "Package Legitimacy Audit": N/A — stdlib `urllib`/`json` only, no new client dep; adding one would break the PyInstaller + stdlib-only contract). |

Existing controls relied upon: the server is the enforcement boundary (Phase 4) — the client never trusts board contents for anything security-relevant; your-best comparison is local (Plan 04 D-11).
