## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | submit_score HMAC gate | mitigate | verify_signature gate per D-02 matrix: invalid sig rejected always; missing sig rejected when REQUIRE_SIGNATURE on |
| T-04-03 | Tampering | submit_score signed payload | mitigate | machine_id + initials + score bound into signed material (D-03) — captured sig cannot be lifted onto another identity/score |
| T-04-06 | Tampering | submit_score score validator | mitigate | sanity ceiling MAX_SCORE = 50_000 (D-01); score>ceiling → 400 |
| T-04-07 | Tampering | permanent-initials lock (D-05) | mitigate | stored initials kept on existing doc; new submission's differing initials never written → blocks tag-swap/impersonation |
| T-04-08 | Spoofing | week bucketing (D-06) | mitigate | week_id computed from datetime.now(timezone.utc) server time only; client-supplied timestamp never read → cannot place a score in a different week |
| T-04-09 | Tampering | replay of valid submission | accept | only-if-higher upsert makes replay near-harmless (D-04); no nonce/timestamp — deliberately accepted residual |
| T-04-10 | Information Disclosure | submit_score logging | mitigate | grace/reject logs never include the secret or full machine_id (RESEARCH V7) |
| T-04-SC | Tampering | dependency installs | accept | This phase installs NO packages (stdlib + already-pinned firebase-admin/functions-framework); requirements.txt unchanged. No package legitimacy gate needed |
