# Requirements: Pac-Man (pacman-firebase) — v1.2 Feels Right

**Defined:** 2026-06-29
**Core Value:** It feels like real Pac-Man — four ghosts with distinct, hand-tuned personalities the player can read and outplay. That behavior is precious and must never silently regress.

## v1.2 Requirements

Game-feel & fairness polish. No new content, no multiplayer. Fairness items change ghost
*outcomes* (positions, who-catches-whom) but **never** ghost *decision logic* (targeting/profiles
stay byte-identical) — treated as sanctioned, isolated, oracle-scoped behavior changes with golden
masters re-blessed on Linux/Docker (the v1.0 Phase-3 box-fix pattern).

### Fairness

<!-- Behavior changes — pinned by the golden net; batch behind one re-bless. -->

- [x] **FAIR-01**: A ghost does not catch the player during a corner turn unless they are genuinely adjacent — collision uses center-to-center distance, not bounding-box overlap, so diagonal corner-kisses no longer register as catches
- [x] **FAIR-02**: The player can pull away from a chasing ghost on a straightaway — Pac-Man moves slightly faster than the ghosts, and the high-tier ghost speed-up is retuned so escape remains possible (never an unbeatable ×2)
- [x] **FAIR-03**: The player can round a corner by inputting the turn slightly before reaching the junction — a pre-turn window registers the queued direction a few pixels early for smooth corner-cutting

### Juice / FX

<!-- Cosmetic — golden-safe unless the frame-hash net hashes pixels (then re-bless). -->

- [x] **FEEL-01**: On death, the player sees Pac-Man play a disintegrate/wedge animation synced to `death.wav` before the round resets
- [x] **FEEL-02**: When the player eats a frightened ghost, the points earned (200/400/800/1600) pop up at the eat location with a brief freeze before play resumes
- [~] **FEEL-03**: ~~When the player eats a ghost, a distinct eat-ghost sound plays~~ — **CUT** (Phase 9): descoped; sourcing/licensing a `.wav` not worth the chore. Eat-ghost audio wiring removed.
- [x] **FEEL-04**: Frightened ghosts blink white as the power-pellet timer is about to expire, signalling when it is no longer safe to chase
- [x] **FEEL-05**: Before each round begins, the player sees a "READY!" beat (text + brief pause) before the ghosts and Pac-Man start moving

## Out of Scope

Explicitly excluded for v1.2. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Scatter/chase wave system | Alters ghost **decision behavior** — the precious never-touch-silently zone; parked, not a feel pass |
| Bonus fruit / extra mazes / modes | Reward & content beats belong to *More Fun*; kept Feels Right lean |
| Real-time multiplayer (1v4) | Needs a play group to validate fun; its own future milestone |
| Level-clear maze flash | Considered, dropped from scope by user during scoping |
| Death screen-shake | Considered; reads as un-arcade, dropped |
| Rising ambient siren, extra-life jingle, fuller intro tune | Audio polish considered; only the eat-ghost sound made the cut to keep it lean |
| Extra life at score threshold, pulsing power-pellets | Considered during scoping; user chose "keep it lean" |
